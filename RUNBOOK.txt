================================================================================
FIRST TIME SETUP  (do this once before running any commands)
================================================================================

--- System requirements ---
  GPU: NVIDIA with 12+ GB VRAM (RTX 3080 / 3090 / 4070 or better)
       Two ~7B models run simultaneously. CPU-only is 50-100x slower.
  RAM: 16 GB minimum, 32 GB preferred
  Disk: 40 GB free  (models 4-8 GB each, results can grow to several GB)

--- Install these tools ---
  Python 3.10+     https://www.python.org/downloads/
  Git              https://git-scm.com/   (Windows: install Git Bash)
  Docker Desktop   https://www.docker.com/get-started
  Ollama           https://ollama.com

--- Clone the repo ---
  git clone <repo_url>
  cd Adversarial_Attacks_on_Code_Large_Language_Models_Using_Reinforcement_Learning

--- Create the Python environment ---
  macOS / Linux:
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -U pip
    pip install -r requirements.txt

  Windows (Git Bash — use this, NOT PowerShell):
    python -m venv .venv
    source .venv/Scripts/activate
    pip install -U pip
    pip install -r requirements.txt

  You will see (.venv) in your prompt when the environment is active.
  Re-activate every time you open a new terminal:
    macOS/Linux:  source .venv/bin/activate
    Windows:      source .venv/Scripts/activate

--- Start Docker and Ollama ---
  Open Docker Desktop. Wait until it says "Docker Engine running".
  Open Ollama (runs in background / system tray).
  Both must be running before any experiment command.

--- Pull the models for your section ---
  Pedro will tell you which section to run. Each section header names the models.

  Section 1 : llama3.1:8b + qwen2.5-coder:7b
  Section 2 : llama3.1:8b + deepseek-coder:6.7b
  Section 3 : llama3.1:8b + codellama:7b
  Section 4 : llama3.1:8b + starcoder2:7b
  Section 5 : tulu3:8b    + qwen2.5-coder:7b
  Section 6 : tulu3:8b    + deepseek-coder:6.7b
  Section 7 : tulu3:8b    + codellama:7b
  Section 8 : tulu3:8b    + starcoder2:7b
  Section 9 : olmo2:7b    + qwen2.5-coder:7b
  Section 10: olmo2:7b    + deepseek-coder:6.7b
  Section 11: olmo2:7b    + codellama:7b
  Section 12: olmo2:7b    + starcoder2:7b

  Pull both models for your section, e.g. for Section 1:
    ollama pull llama3.1:8b
    ollama pull qwen2.5-coder:7b

--- Smoke test (verify everything works) ---
  macOS / Linux:
    PYTHONPATH=V3 .venv/bin/inspect eval V3/adversarial_attack.py@adversarial_code_llm \
      --model ollama/llama3.1:8b \
      -T benchmark=adversarial_code_buggy -T policy_mode=random_choice \
      -T mutation_strategy=react -T experiment_mode=one_shot \
      -T use_llm_judge=True -T target_model=ollama/qwen2.5-coder:7b \
      -T max_iterations=1 --max-samples 1 --limit 1

  Windows (Git Bash):
    PYTHONPATH=V3 .venv/Scripts/inspect eval V3/adversarial_attack.py@adversarial_code_llm \
      --model ollama/llama3.1:8b \
      -T benchmark=adversarial_code_buggy -T policy_mode=random_choice \
      -T mutation_strategy=react -T experiment_mode=one_shot \
      -T use_llm_judge=True -T target_model=ollama/qwen2.5-coder:7b \
      -T max_iterations=1 --max-samples 1 --limit 1

  Passes if: a folder appears under results/ and no Python exceptions in console.
  If it fails, send Pedro a screenshot before continuing.

--- Rules for running ---
  * Run from: cd <project_folder>  (same folder as this file)
  * One command at a time — never run two at once on the same machine (GPU memory)
  * Each experiment command takes 2-12 hours. Leave the machine on and let it finish.
  * If a command crashes, run the same command again. Results are additive.
  * All 4 bandit algorithms (ucb1, thompson, klucb, exp3) are implemented and ready
  * Do NOT delete the weights/ folder between runs

--- Results storage workflow ---
  After EACH experiment command completes, run the store command shown below it.
  This saves the .eval file and results/ into stored_results/<section>/<run>/.
  results/ is left empty and ready for the next run.

  Final layout when all sections are done:
    stored_results/
      section-1/
        random-llama31-qwen-adversarial_code_buggy/
          *.eval
          results/
        random-llama31-qwen-cubert_wbo/
          *.eval
          results/
        react-llama31-qwen-adversarial_code_buggy/
        ...
      section-2/
        random-llama31-deepseek-adversarial_code_buggy/
        ...
      ...
      section-13/
        ...

  macOS / Linux:

Pedro merges all stored_results/ subfolders into one directory, merges
weights/ files, then runs:

  bash V3/scripts/aggregate_and_plot.sh

This produces Resume files, aggregated tables, and all plots for the thesis.
